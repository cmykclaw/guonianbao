## ADDED Requirements

### Requirement: 折叠面板分类展示亲戚关系
系统 SHALL 使用 van-collapse 折叠面板组件，将亲戚关系按分类展示为 4 个可展开的折叠项。

#### Scenario: 默认展开第一项
- **WHEN** 页面加载
- **THEN** 折叠面板默认展开"堂表迷阵"分类

#### Scenario: 展开指定分类
- **WHEN** 用户点击某个折叠项标题
- **THEN** 该分类展开显示其中的亲戚关系，其他分类收起

#### Scenario: 收起分类
- **WHEN** 用户再次点击已展开的折叠项
- **THEN** 该分类收起

### Requirement: 四类亲戚关系数据展示
系统 SHALL 展示以下四类亲戚关系：
1. 堂表迷阵（同辈）
2. 长辈配偶（最怕叫错）
3. 混淆晚辈（发红包必看）
4. 婚后专属（地狱级难度）

#### Scenario: 显示堂表迷阵分类
- **WHEN** 展开"堂表迷阵"分类
- **THEN** 显示爸爸的兄弟的孩子、爸爸的姐妹的孩子、妈妈的兄弟的孩子、妈妈的姐妹的孩子

#### Scenario: 显示长辈配偶分类
- **WHEN** 展开"长辈配偶"分类
- **THEN** 显示姑父、舅妈、姨夫、伯母、婶婶等

#### Scenario: 显示复杂晚辈分类
- **WHEN** 展开"混淆晚辈"分类
- **THEN** 显示侄子、侄女、外甥、外甥女等

#### Scenario: 显示婚后专属分类
- **WHEN** 展开"婚后专属"分类
- **THEN** 显示连襟、妯娌、大舅哥、小姨子等

### Requirement: 搜索过滤功能
系统 SHALL 支持在折叠面板中进行搜索过滤。

#### Scenario: 搜索匹配
- **WHEN** 用户在搜索框输入关键词
- **THEN** 折叠面板中只显示匹配的亲戚关系

### Requirement: 保留原有核心逻辑
系统 SHALL 保留原有的搜索框、isAILoading、AI 计算、watch 防抖等核心逻辑不变。

#### Scenario: 核心逻辑保留
- **WHEN** 页面功能测试
- **THEN** 搜索、AI 计算等功能正常工作
